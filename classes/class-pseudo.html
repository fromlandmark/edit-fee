CSS Pseudo-seletores e Pseudo-estados

Uma análise técnica dos pseudo-seletores e pseudo-estados em CSS, detalhando as suas propriedades, comportamentos e aplicações em implementações de interfaces web modernas.

Definição e Scope

Os pseudo-seletores e pseudo-estados são extensões da sintaxe de seletores CSS que permitem selecionar elementos baseados em critérios que transcendem a estrutura do documento DOM. Estes mecanismos oferecem seleção contextual, interativa e posicional sem necessidade de marcação adicional.

Categorização Funcional

Existem duas categorias principais de pseudo-seletores no CSS:

Pseudo-classes

Selecionam elementos baseados em estados, posições ou atributos que não são explicitamente definidos na estrutura do documento. São identificadas por dois-pontos (:) seguidos do identificador da pseudo-classe.

Pseudo-elementos

Permitem estilizar partes específicas de um elemento ou gerar conteúdo que não existe explicitamente no DOM. São identificados por dois dois-pontos (::) seguidos do identificador do pseudo-elemento.

Pseudo-classes

Pseudo-classes de Estado Interativo

Aplicam-se a elementos baseados na interação do utilizador.

:hover

Seleciona um elemento quando o ponteiro está sobre ele.

a:hover {
  color: #ff5722;
  text-decoration: underline;
}

:active

Seleciona um elemento quando está a ser ativado pelo utilizador (tipicamente durante o clique).

button:active {
  transform: translateY(2px);
  background-color: #0056b3;
}

:focus

Seleciona um elemento quando recebe o foco (via teclado ou rato).

input:focus {
  border-color: #4a90e2;
  box-shadow: 0 0 0 3px rgba(74, 144, 226, 0.3);
}

:focus-visible

Seleciona um elemento quando recebe o foco e o User Agent determina que deve ser visualmente indicado.

button:focus-visible {
  outline: 2px solid #4a90e2;
  outline-offset: 2px;
}

:focus-within

Seleciona um elemento que recebeu o foco ou possui um descendente que recebeu o foco.

form:focus-within {
  background-color: #f8f9fa;
}

Pseudo-classes de Estado de Input

Aplicam-se especificamente a elementos de formulário baseados no seu estado.

:checked

Seleciona elementos de input do tipo radio, checkbox ou option quando selecionados.

input[type="checkbox"]:checked {
  background-color: #4caf50;
}

:disabled e :enabled

Selecionam elementos de formulário baseados no seu estado de disponibilidade.

input:disabled {
  opacity: 0.6;
  cursor: not-allowed;
}

input:enabled {
  background-color: #ffffff;
}

:valid e :invalid

Selecionam elementos de formulário baseados na validação do seu valor segundo restrições especificadas.

input:valid {
  border-color: #4caf50;
}

input:invalid {
  border-color: #f44336;
}

:required e :optional

Selecionam elementos de formulário baseados na sua obrigatoriedade.

input:required {
  border-left: 4px solid #ff9800;
}

input:optional {
  border-left: 1px solid #e0e0e0;
}

Pseudo-classes Estruturais

Selecionam elementos baseados na sua posição na estrutura do documento.

:first-child e :last-child

Selecionam elementos que são o primeiro ou último filho do seu parente.

li:first-child {
  border-top-left-radius: 4px;
  border-top-right-radius: 4px;
}

li:last-child {
  border-bottom-left-radius: 4px;
  border-bottom-right-radius: 4px;
}

:nth-child() e :nth-last-child()

Selecionam elementos baseados na sua posição entre os irmãos, aceitando fórmulas algébricas.

tr:nth-child(even) {
  background-color: #f2f2f2;
}

li:nth-child(3n+1) {
  clear: left;
}

/* Seleciona o terceiro elemento a contar do fim */
li:nth-last-child(3) {
  color: #e91e63;
}

:nth-of-type() e :nth-last-of-type()

Similares a :nth-child(), mas contabilizam apenas elementos do mesmo tipo.

p:nth-of-type(2) {
  font-style: italic;
}

img:nth-of-type(odd) {
  float: left;
  margin-right: 10px;
}

:only-child e :only-of-type

Selecionam elementos que são filhos únicos do seu parente, no geral ou do seu tipo.

li:only-child {
  list-style: none;
}

h2:only-of-type {
  border-bottom: 2px solid #333;
}

:empty

Seleciona elementos que não têm filhos (incluindo nós de texto).

div:empty {
  display: none;
}

Pseudo-classes de Negação e Condicionais

:not()

Seleciona elementos que não correspondem ao seletor especificado.

input:not([type="submit"]) {
  border: 1px solid #ddd;
}

p:not(:first-child) {
  margin-top: 1em;
}

:is() e :where()

Agrupamento de seletores, com :is() mantendo a especificidade do seletor mais forte e :where() tendo especificidade zero.

/* Equivalente a header p, main p, footer p */
:is(header, main, footer) p {
  line-height: 1.6;
}

/* Mesmo comportamento mas com especificidade zero */
:where(header, main, footer) p {
  font-size: 1rem;
}

:has()

Seleciona elementos que contêm pelo menos um elemento que corresponde ao seletor especificado.

/* Parágrafos que contêm imagens */
p:has(img) {
  display: flex;
  align-items: center;
}

/* Artigos que contêm um cabeçalho h2 seguido de um parágrafo */
article:has(h2 + p) {
  padding: 20px;
}

Pseudo-elementos

Permitem estilizar partes específicas de um elemento ou gerar conteúdo que não existe explicitamente no DOM.

::before e ::after

Criam elementos antes ou depois do conteúdo de um elemento, utilizados frequentemente com a propriedade content.

blockquote::before {
  content: '"';
  font-size: 2em;
  color: #999;
}

a[href^="http"]::after {
  content: " ↗";
  color: #999;
  font-size: 0.8em;
}

::first-letter e ::first-line

Selecionam a primeira letra ou a primeira linha de texto dentro de um elemento.

p::first-letter {
  font-size: 2em;
  font-weight: bold;
  float: left;
  margin-right: 0.25em;
}

p::first-line {
  font-variant: small-caps;
}

::selection

Seleciona a parte de um elemento que está selecionada pelo utilizador.

::selection {
  background-color: #b3d4fc;
  color: #000;
  text-shadow: none;
}

::placeholder

Seleciona o texto indicativo (placeholder) dentro de um elemento input.

input::placeholder {
  color: #999;
  font-style: italic;
}

::marker

Seleciona o marcador de um item de lista (como bullets ou números).

li::marker {
  color: #4a90e2;
  font-weight: bold;
}

Regras de Especificidade e Cascade

A especificidade dos pseudo-seletores segue as regras padrão do CSS:

  • Pseudo-classes têm a mesma especificidade que classes normais (0,0,1,0);
  • Pseudo-elementos têm a mesma especificidade que elementos normais (0,0,0,1);
  • O seletor :not() não adiciona especificidade, mas os seus argumentos são considerados;
  • O seletor :where() tem especificidade zero, independentemente dos seus argumentos;
  • O seletor :is() assume a maior especificidade dos seus argumentos.

Casos de Uso Comuns

Estilização Interativa

/* Botão com efeitos de hover e active */
.button {
  background-color: #4a90e2;
  color: white;
  padding: 10px 15px;
  border: none;
  border-radius: 4px;
  transition: all 0.2s ease;
}

.button:hover {
  background-color: #3a80d2;
  transform: translateY(-2px);
  box-shadow: 0 2px 5px rgba(0,0,0,0.2);
}

.button:active {
  transform: translateY(0);
  box-shadow: 0 1px 2px rgba(0,0,0,0.2);
}

Validação de Formulários

/* Estilização de campos baseada na validação */
input {
  border: 1px solid #ddd;
  padding: 8px 12px;
  border-radius: 4px;
  transition: all 0.2s ease;
}

input:focus {
  border-color: #4a90e2;
  box-shadow: 0 0 0 3px rgba(74, 144, 226, 0.3);
  outline: none;
}

input:valid {
  border-color: #4caf50;
}

input:invalid {
  border-color: #f44336;
}

input:required {
  background-image: url('asterisk.svg');
  background-repeat: no-repeat;
  background-position: right 8px center;
  background-size: 8px;
  padding-right: 24px;
}

Elementos Decorativos

/* Adição de elementos decorativos sem alterar o HTML */
.card {
  position: relative;
  padding: 20px;
  border-radius: 8px;
  background-color: white;
}

.card::before {
  content: '';
  position: absolute;
  top: 0;
  left: 0;
  width: 100%;
  height: 4px;
  background: linear-gradient(to right, #4a90e2, #9b59b6);
  border-radius: 8px 8px 0 0;
}

.card.featured::after {
  content: 'Em destaque';
  position: absolute;
  top: -10px;
  right: 10px;
  background-color: #e91e63;
  color: white;
  padding: 2px 8px;
  border-radius: 3px;
  font-size: 12px;
}

Estilização Tabular e Listagens

/* Estilização zebrada em tabelas */
tr:nth-child(even) {
  background-color: #f9f9f9;
}

/* Destacar a primeira linha de uma tabela */
tr:first-child {
  background-color: #f2f2f2;
  font-weight: bold;
}

/* Estilização de listas */
ul li:not(:last-child) {
  margin-bottom: 8px;
}

ol li::marker {
  color: #4a90e2;
  font-weight: bold;
}

Considerações de Compatibilidade

Alguns pseudo-seletores mais recentes como :has(), :is() e :where() podem não ter suporte universal. É recomendada a verificação da compatibilidade antes da implementação e a consideração de alternativas para browsers não suportados.

Metodologia de Implementação

A aplicação eficaz de pseudo-seletores requer análise estratégica da estrutura do documento e interações do utilizador. Recomenda-se a utilização de pseudo-elementos para estilização decorativa e de pseudo-classes para estados interativos, minimizando a necessidade de classes JavaScript-dependentes e mantendo a separação entre estrutura e apresentação.
